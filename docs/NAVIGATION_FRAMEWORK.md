# SDD-003

# Navigation Framework

Version: 1.0

Status: Draft

---

# Purpose

This document defines navigation throughout OEP Studio.

Navigation shall remain consistent across all future Studios.

---

# Primary Navigation

Studio shall use a permanent left navigation rail.

The initial navigation items are:

* Dashboard
* Repository
* Objects
* Relationships
* Search
* Graph
* Validation
* Packages
* Settings

Future Studios may add additional items without changing existing navigation.

---

# Workspace

Only one primary workspace is visible at a time.

Changing navigation updates the workspace.

Navigation shall not create floating windows.

---

# Status Bar

The bottom status bar shall display:

* Repository
* Runtime State
* Validation Status
* Foundation Version

Future versions may display synchronization and AI status.

---

# Breadcrumbs

Workspace navigation shall support breadcrumbs for future deep hierarchies.

---

# Keyboard Navigation

Every navigation item shall have a keyboard shortcut.

Command Palette integration is defined by a future specification.

---

# Engineering Principle

Navigation shall make engineering knowledge immediately discoverable.

Users should never wonder where functionality resides.
