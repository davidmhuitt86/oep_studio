# SDD-002

# OEP Studio Design Language

Version: 1.0

Status: Draft

---

# Purpose

This document defines the visual identity and interaction language of OEP Studio.

The design language shall prioritize clarity, density of information, and engineering productivity over visual decoration.

---

# Design Philosophy

Studio shall resemble professional engineering software rather than consumer applications.

Primary goals:

* Fast navigation
* High information density
* Consistent interaction
* Minimal visual distraction

---

# Theme

Dark theme is the primary supported appearance.

Light theme may be introduced in a future release.

---

# Color Philosophy

Colors communicate state rather than decoration.

Examples:

* Blue — selection
* Green — success
* Yellow — warning
* Red — error
* Gray — inactive

Accent colors shall be used sparingly.

---

# Typography

Use a single sans-serif type family throughout the application.

Hierarchy shall be established through weight and size rather than multiple font families.

Monospaced fonts shall be used for:

* IDs
* CLI output
* JSON
* Technical data

---

# Layout

Studio shall use a panel-based layout.

Components include:

* Navigation Rail
* Workspace
* Status Bar
* Toolbar
* Dialogs

Future specifications define dockable panels.

---

# Icons

Icons shall communicate engineering concepts.

Avoid decorative icons.

Labels shall accompany icons until usability testing demonstrates otherwise.

---

# Motion

Animations shall be subtle.

Animation shall never delay engineering workflows.

---

# Engineering Principle

The interface exists to expose engineering knowledge.

Visual design shall always support engineering productivity.
